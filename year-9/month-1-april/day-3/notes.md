## Active Recall

**Q1: What is the difference between embedding a video using `<video>` directly versus using `<iframe>`, and when would you use each one?**

A: `<video>` is used to embed a video file stored locally or on your own server, referencing it with a file path or direct URL. `<iframe>` is used to embed content hosted on an external platform like YouTube directly onto 
    your page without storing the file yourself.

---

**Q2: What does the `src` attribute do on a `<video>` tag, and what happens if you leave it out?**

A: The `src` attribute sets the source for the media being displayed, either as a URL or a local file path. If you leave it out, the `<video>` element renders an empty frame and no media plays.

---

**Q3: What is an `<iframe>` and what does the `src` attribute point to when embedding a YouTube video?**

A: An `<iframe>` is an inline frame used to embed external content inside a webpage. When embedding a YouTube video, the `src` attribute points to the custom embed URL format — `youtube.com/embed/videoID` — not the standard
   watch URL.

---

**Q4: What HTML element or method did you use to display the heart icon, and why can't you just type the symbol directly in most cases?**

A: I used the `<svg>` element to create the heart icon. SVG scales perfectly across different screen resolutions and devices without distorting, unlike typing the symbol directly which can render inconsistently across 
   browsers and screen sizes.
